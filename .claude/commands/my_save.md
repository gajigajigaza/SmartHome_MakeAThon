# my_save

이번 세션에 일어난 작업을 검토하고, 두 저장소에 적절히 분산 저장해줘.

## 수행 절차

### 1. 세션 작업 검토
이번 세션에서 실제로 일어난 일을 추려낸다:
- 새로 작성/수정한 파일
- 실행한 명령과 결과
- 내린 결정과 그 이유
- 새로 발견한 사실, 통찰
- 미완 작업과 이어갈 항목

### 2. 두 저장소에 분산

#### (A) CLAUDE.md : 프로젝트 규칙으로 승격할 것만
이런 것만 추가:
- 앞으로 반복해서 적용될 규칙
- 새로 확립된 컨벤션, 금기사항
- 빌드/테스트 명령 추가, 변경
- 아키텍쳐적 결정

**추가 안함**
- 이미 존재하는 내용과 중복
- 일회성, 오늘만의 맥락

기존 섹션 구조를 존중하고, 적합한 헤더 아래에 추가한다. 추가할 것이 없으면 수정하지 말고 그 이유를 사용자에게 알린다.

#### (B) plans/daily-YYYY-MM-DD.md : 오늘 세션의 작업 기록 전부
오늘 세션에서 있었던 일을 기록한다:
- 새로 작성/수정한 파일 (경로 + 무엇을/왜 바꿨는지)
- 실행한 명령과 결과 (성공/실패 여부, 핵심 출력만 요약)
- 내린 결정과 그 이유 (특히 여러 선택지 중 이걸 고른 이유)
- 새로 발견한 사실, 통찰
- 미완 작업과 이어갈 항목 (다음 세션에서 바로 이어갈 수 있을 만큼 구체적으로)

날짜는 오늘 날짜(YYYY-MM-DD) 기준으로 `plans/daily-YYYY-MM-DD.md` 파일명을 정한다. `plans/` 디렉토리가 없으면 새로 만든다. 같은 날짜 파일이 이미 있으면 덮어쓰지 말고 이어서 append한다(같은 날 여러 세션 기록이 한 파일에 누적되는 동시에 작업별로 분리하여 작성한다).

### 3. 완료 후 보고
CLAUDE.md에 무엇을 추가했고(또는 왜 추가하지 않았고), daily 파일에 무엇을 기록했는지 간단히 요약해서 사용자에게 알린다.
